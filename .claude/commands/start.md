# /start - プロジェクト初期化コマンド

タスク: **$ARGUMENTS**

## ステップ1: ブランチ作成・移動

現在のブランチから新しいfeatureブランチを作成します。

**注意**: 日本語の引数が与えられた場合は、タスク内容を理解して適切な英語ブランチ名に変換します。

### ブランチ名の推論と作成
タスク「$ARGUMENTS」の内容を分析し、適切な英語ブランチ名を推論してください。
以下の方針に従ってブランチ名を決定してください：

- **簡潔性**: 2-3語以内、ハイフン区切り
- **明確性**: 実装内容が分かる名前
- **一貫性**: 既存のブランチ命名規則に従う
- **英語表記**: 日本語は英語に変換

例：
- "AppShell追加" → "app-shell"
- "ユーザー認証機能" → "user-authentication"  
- "管理画面改善" → "admin-improvements"
- "ポーカーゲーム機能" → "poker-game-features"

推論したブランチ名で以下を実行：
!git checkout -b "feature/[推論したブランチ名]"

## ステップ2: 対話形式での仕様作成

以下の質問に答えて、詳細な仕様書を作成しましょう：

### 1. タスクの目的・背景
**質問**: このタスクはなぜ必要ですか？どのような問題を解決しますか？

### 2. 具体的な要件
**質問**: 
- どのような機能を実装しますか？
- ユーザーはどのように操作しますか？
- どのような入力・出力が期待されますか？

### 3. 技術的制約・考慮事項
**質問**:
- 既存のコードベースで考慮すべき点はありますか？
- パフォーマンス要件はありますか？
- セキュリティ上の注意点はありますか？

### 4. 実装方針
**質問**:
- どのようなアプローチで実装しますか？
- 使用する技術・ライブラリはありますか？
- ファイル構成はどうしますか？

### 5. テスト戦略
**質問**:
- どのようなテストが必要ですか？
- テストケースを具体的に教えてください。
- エッジケースはありますか？

## ステップ3: 仕様書ファイル生成

以下の情報を元に仕様書を作成します：

- **タスク名**: $ARGUMENTS
- **作成日時**: $(date '+%Y-%m-%d %H:%M:%S')
- **ブランチ名**: feature/[推論したブランチ名]

仕様書は `specs/$(date '+%Y%m%d')-[推論したブランチ名].md` に保存されます。

---

**次のステップ**: 
1. 上記の質問に回答してください
2. 仕様書を確認・編集してください  
3. 実装を開始してください
